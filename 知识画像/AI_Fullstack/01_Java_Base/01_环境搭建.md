## 🧠 思维模型：Java 是如何工作的？

在开始安装之前，你需要理解每一个组件的**角色**（Role）：

### 1. 为什么需要 JDK？(The Toolbox)
你写的代码 (`.java`) 本质上只是文本文件，电脑看不懂。
你需要一个**翻译官**。
*   **JDK (Java Development Kit)**: 这是一个**工具箱**。
    *   `javac`: **编译器 (Compiler)**。它负责把你的“人话”(源码) 翻译成“中间语”(字节码 .class)。
    *   `java`: **运行器 (Runtime)**。它负责启动虚拟机来执行这些字节码。
*   **JRE vs JDK**: JRE 只是环境（给用户用的），JDK 包含编译器（给程序员用的）。所以你需要 JDK。

### 2. 什么是 JVM？(The Magic Box)
**"Write Once, Run Anywhere" (一次编写，到处运行)** 是 Java 的灵魂。
*   **原理**: Java 不直接在这个真实的 CPU 上跑，而是在一个软件模拟的**虚拟 CPU (JVM)** 上跑。
*   **类比**: JVM 就像是一个**通用的游戏模拟器**。不管是 Windows 还是 Mac，只要装了这个模拟器，就能玩同一个游戏卡带 (`.class` 文件)。

## ✅ 实操清单 (Action Items)

### 1. 搭建地基 (JDK Setup)
> **Why?** 没有地基（编译器和虚拟机），高楼（代码）无法建立。

- [ ] **下载 JDK 21**: 获取官方工具箱。
- [ ] **配置环境变量**: 
    - *思维*: 为什么要配 `PATH`？是为了让操作系统在任何文件夹下都能找到 `javac` 这个命令，而不用每次都输全路径。
- [ ] **验证**: `java -version` (确保虚拟机就位), `javac -version` (确保编译器就位)。

### 2. 准备图纸 (IDE Setup)
> **Why?** 虽然用记事本也能写代码，但 IDEA 就像是带自动纠错、自动排版的**智能绘图桌**。

- [ ] 安装 IntelliJ IDEA
- [ ] 创建新项目: 这一步是在通过 IDE 初始化项目的目录结构。

### 3. 第一座建筑 (Hello World)
> **Why?** `public static void main` 看起来很晕？
> *   `class Hello`: Java 中**万物皆对象 (Class)**。代码必须住在类里。
> *   `main`: 这是程序的**正门**。JVM 启动时只认这个门。

- [ ] 编写代码:
    ```java
    public class Hello {
        // 程序入口
        public static void main(String[] args) {
            System.out.println("Hello Java"); // 向控制台输出
        }
    }
    ```
- [ ] 运行: 观察控制台输出。

## 📝 深度思考 (Deep Dive)
- **Q: 为什么 Java 要分编译 (`javac`) 和运行 (`java`) 两步？Python 好像直接运行就可以？**
    - *A: 编译期是第一道防线。Java 在你运行之前就会检查所有类型错误（静态类型），这在几百万行代码的大项目中能救命。Python 是边跑边检查（动态类型），写起来爽，但大规模协作容易出错。*
